3. Payment Plan & Schedule
Create Payment Plan
/Payment/CreatePaymentPlan
ProjectID, PlanName, TotalAmount (PKR/USD)
DurationMonths, Frequency, TokenAmount
↓
Auto-Calculation
distributable = TotalAmount − TokenAmount
installmentAmount = distributable ÷ N
Last installment absorbs rounding remainder
↓
PaymentPlan Table
PlanID (auto), ProjectID, TotalAmount, TotalAmountUSD
↓
PaymentSchedule Table
ScheduleID per installment (Token #0 + #1…N)
DueDate spaced by Frequency
↓
Assigned to Customer
PlanID stored on Customers.PlanID
Customer pays against each ScheduleID
FrequencyDue Date Interval
Monthly+1 month
Quarterly+3 months
Half Yearly+6 months
Yearly+12 months

Validation Rule
Token + all installments ≤ TotalAmount
SurchargeRate stored ÷ 100 (form sends %)
USD auto-calculated if only PKR given (exchange rate from Config)

5. Customer Payment
Prerequisite
Customer exists with PlanID → PaymentSchedule installments
↓
Record Payment
/Payment/RecordPayment
Select CustomerID → AJAX loads outstanding installments
Select ScheduleID, enter Amount, Method, ReferenceNo
↓
Validation
Amount > 0 and ≤ Outstanding balance for that installment
Outstanding = Schedule.Amount − SUM(existing payments)
↓
Payments Table
PaymentID, ScheduleID (FK), CustomerID (FK)
Amount, Method, ReferenceNo, Status
AuditStatus = Pending (default)
↓
Status Auto-Set
Paid — if payment covers full outstanding
Partially Paid — otherwise
FeatureNotes
Single PaymentOne installment, one record
Multiple (Bulk)JSON payload; each row = separate Payment record; sum must equal stated total
PenaltyExtra charge added to customer (Penalties table)
WaiverDiscount/forgiveness (Waivers table, records ApprovedBy)
Delete PaymentAdmin only; permanent; logged
6. Payment Audit / Verification
Payment Record
AuditStatus = Pending (default on every new payment)
↓
Auditor Reviews
/PaymentAudit/Audit
Enters remarks, chooses Approved or Declined
↓
Audit Fields Updated (no amount change)
AuditStatus, AuditedBy, AuditedAt, AuditRemarks
↓
Reset (Admin only)
Clears audit fields → AuditStatus = Pending for re-review
ActionPermissionEffect
ApproveEditStamps Approved + auditor info. Payment unchanged.
DeclineEditStamps Declined. Payment unchanged.
ResetAdminClears audit stamp; not logged to ActivityLog.

9. Refund
Prerequisite
Customer with Payments not already refunded
↓
Refund Create
/Refund/Create
Search Customer → AJAX returns eligible payments
Select one or more Payments, enter DeductionAmount, Reason
↓
Auto-Calculation
PaidAmount = SUM of selected payments
RefundedAmount = PaidAmount − DeductionAmount (min 0)
↓
Refund Table
RefundID (RFD0000001), WorkflowStatus = Initiated
SelectedPaymentIDs stored as JSON array
↓
Initiated → Approved (payments deleted) / Declined (no change)
On Approval
All SelectedPaymentIDs permanently deleted from Payments table
Each deletion logged in ActivityLog
ActionPermissionEffect
ApproveAdminDeletes selected payments. Irreversible.
DeclineEditStatus → Declined. Payments untouched.

Refund FieldSource
PaidAmountSUM of selected Payments.Amount
DeductionAmountUser input (default 0)
RefundedAmountAuto: PaidAmount − DeductionAmount
SelectedPaymentIDsJSON array; deleted on approval
